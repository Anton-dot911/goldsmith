# ТЗ №5 — Golden Dataset Builder (внутрішній інструмент)

**Кодова назва:** Goldsmith
**Тип:** веб-інструмент для особистого користування
**Пріоритет:** Етап 1 (паралельно з DocFlow, коли з'являться перші дані)
**Оцінка тривалості:** 3–4 дні

---

## 1. Призначення

Усі три портфоліо-проєкти мають наскрізну вимогу: golden datasets для evals (екстракція полів, routing, SQL-питання, пер-агентні набори). Створення таких датасетів вручну (JSON у редакторі) — повільне і схильне до помилок. Goldsmith — швидкий інтерфейс розмітки: приклад → еталонна відповідь → експорт у формат, який споживають eval-скрипти.

## 2. Функціональні вимоги

### MVP
- **F1. Датасети:** створення датасету з назвою, описом і **схемою еталона** (JSON Schema, вставляється або обирається з пресетів: extraction / routing / qa / classification).
- **F2. Додавання прикладів:** ручний ввід; масовий імпорт входів (CSV/JSONL/тексти файлами); для DocFlow — завантаження документа як входу.
- **F3. Розмітка:** двопанельний режим — зліва вхід (текст/рендер документа), справа форма еталона, згенерована зі схеми датасету. Валідація за схемою при збереженні.
- **F4. AI-передрозмітка:** кнопка "Draft with AI" — Claude пропонує еталон, людина виправляє і підтверджує (розмітка у 3–5 разів швидша, а виправлення — сигнал складних кейсів). Кожен приклад позначений: `ai_drafted+human_verified | human_only`.
- **F5. Теги і категорії:** довільні теги на прикладах (для метрик по зрізах: "скан", "багатосторінковий", "mixed-route"...).
- **F6. Версіонування:** датасет має версії; експорт завжди з номером версії; приклади не видаляються, а деактивуються.
- **F7. Експорт:** JSONL (канонічний формат: `{id, input, expected, tags[], version}`) — файлом і через простий read-ендпоінт для CI.

### v1.1
- F8. Режим перегляду розбіжностей: імпорт результатів eval-прогону → side-by-side expected vs actual, швидке виправлення еталонів-помилок.
- F9. Дублікат-детектор (embedding similarity) при додаванні прикладів.

### Out of scope
Мультикористувацька розмітка, ролі, аудит — це особистий інструмент.

## 3. Технічні рішення

- React + Vite + TS + Tailwind (Netlify) + Supabase (таблиці `dataset`, `dataset_version`, `example`; Storage для файлів-входів).
- AI-передрозмітка — через Netlify Function (ключ на сервері).
- Форма з JSON Schema: react-jsonschema-form або власний рендерер для 4 пресетів (обрати простіше).

## 4. Definition of Done

- [ ] Датасет на 40 прикладів для DocFlow створено через Goldsmith швидше ніж за 2 год
- [ ] Eval-скрипт DocFlow читає експорт Goldsmith без ручних правок
- [ ] AI-передрозмітка працює і позначає походження прикладів
